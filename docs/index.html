Chittagong Diabetes Data Analysis

Data cleaning, regional distribution, age-group patterns, and clinical data preparation

Author

Miskatul Anwar

Published

2026-06-05

Overview

This project documents a reproducible workflow for preparing and analyzing diabetes patient data from the Chittagong region. The current version of the project focuses on two major stages: cleaning and standardizing the raw clinical dataset, then analyzing the cleaned records by Thana, City Corporation classification, and age group.

The analysis pipeline is designed to support transparent reporting and future extension. Dataset paths are managed through environment variables, cleaned records are exported for downstream analysis, and location-based summaries are generated from the standardized dataset.

Project Scope

The project currently covers the following analytical areas:

  • Cleaning raw diabetes patient records and removing duplicate entries.
  • Updating fasting glucose values from a supplementary dataset using patient ID.
  • Standardizing mixed-format clinical variables into analysis-ready numeric fields.
  • Removing records with missing key analytical fields such as Fasting, Age, and Thana where required.
  • Standardizing Thana names to reduce spelling and administrative-label inconsistencies.
  • Separating records into City Corporation and outside City Corporation groups.
  • Creating age groups using the threshold <=30 and >30.
  • Calculating population counts, Thana-wise distributions, and percentage breakdowns.
  • Verifying whether the final regional grouping accounts for the full filtered dataset.

Analysis Workflow

The project is organized into a sequential workflow. Each stage produces outputs that support the next stage of analysis.

Data Cleaning and Standardization

The first stage prepares the raw diabetes dataset for analysis. It loads the primary dataset and supplementary fasting-value dataset from environment-defined paths, removes duplicate columns and duplicate patient records, updates fasting values, standardizes numeric fields, and exports the cleaned dataset.

Key tasks in this stage include:

  • Loading the raw Excel dataset from DATASET_PATH.
  • Removing duplicate columns.
  • Deduplicating records by ID, keeping the last available record.
  • Loading the supplementary file from NEW_PATH.
  • Updating the Fasting column by matching records on ID.
  • Converting BMI, Weight (KG), and Height (CM) to numeric values.
  • Extracting numeric values from mixed-format Fasting entries.
  • Removing records with missing fasting glucose or age values.
  • Saving the cleaned dataset to CLEANED_DATASET_PATH.

The complete documentation for this stage is available in:

data_cleaning_and_standardization.qmd

Thana-Level Population Distribution and Age Group Analysis

The second stage uses the cleaned dataset to examine the regional distribution of patient records. It focuses on Thana-level classification, City Corporation grouping, age-group segmentation, and percentage-based summaries.

Key tasks in this stage include:

  • Loading the cleaned dataset from CLEANED_DATASET_PATH.
  • Removing Thanas outside the selected analysis scope.
  • Dropping records with missing Thana values.
  • Mapping alternative Thana names to standardized labels.
  • Defining Thanas inside and outside the City Corporation.
  • Creating the Age Group variable using <=30 and >30 categories.
  • Calculating total population records after filtering.
  • Summarizing records inside the City Corporation.
  • Summarizing records outside the City Corporation.
  • Calculating Thana-level and age-group percentages.
  • Running a final verification check to identify unmapped or unclassified records.

The complete documentation for this stage is available in:

thana_population_distribution_analysis.qmd

Data Sources and Environment Variables

This project avoids hard-coded file paths. The workflow expects file locations to be provided through environment variables.

Required variables:

DATASET_PATH
NEW_PATH
CLEANED_DATASET_PATH

These variables should be defined in a .env file or made available directly in the execution environment before rendering the project files.

Current Dataset Preparation Logic

The latest data preparation workflow uses the following assumptions and rules:

  • ID is treated as the unique patient identifier for deduplication.
  • If multiple records share the same ID, the last occurrence is retained.
  • The supplementary dataset is used to update fasting glucose values in the primary dataset.
  • Fasting glucose values may contain units or mixed text, so the first numeric value is extracted.
  • Records without valid fasting glucose values are removed from the cleaned dataset.
  • Records without age values are removed from the cleaned dataset.
  • Thana names are standardized before regional analysis.
  • Records without valid Thana values are removed before Thana-level analysis.
