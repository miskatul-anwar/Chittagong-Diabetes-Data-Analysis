5.1 City Corporation

This section visualizes the distribution of diabetes patients across selected thanas of Chattogram City Corporation. The map uses GADM level-3 administrative boundary data from a GeoPackage file and joins it with manually entered thana-level diabetes percentage data.

The spatial boundary file path is loaded from the .env file using the GPKG_PATH environment variable. This keeps the source code cleaner and avoids hard-coding the data path directly inside the analysis script.

The workflow follows these main steps:

  1. Load the required Python libraries.
  2. Read the GeoPackage boundary file using geopandas.
  3. Create a tabular dataset containing diabetes percentages by thana.
  4. Clean and standardize thana names for reliable joining.
  5. Join the diabetes dataset with the spatial boundary dataset.
  6. Classify diabetes percentages into fixed percentage intervals.
  7. Apply a red choropleth color palette, starting from a darker red shade.
  8. Save the final map as a PDF under the report/ directory.
  9. Preview the generated map directly with display(fig).

The diabetes percentages are classified into the following groups:

  • 1–3%
  • 3–5%
  • 5–7%
  • 7–9%
  • 9–11%
  • 11–13%
  • >13%

A custom red color palette is used for the choropleth map. Unlike very pale choropleth palettes, the first class begins with a visibly darker red shade so that even low-percentage areas remain clearly visible on the map.

def clean_key(x):
    """
    Standardize text for joining:
    lowercase + remove all non-letter characters.
    """
    if pd.isna(x):
        return None

    return re.sub(r"[^a-z]", "", str(x).lower())


def strip_admin_suffix(x):
    """
    Remove common administrative suffixes from place names.
    """
    if pd.isna(x):
        return None

    pattern = r"\s+(Sadar Upazila|Sadar|Upazila|Thana|PS)$"

    return re.sub(pattern, "", str(x), flags=re.IGNORECASE).strip()

5.2 City Corporation with Age Group Composition

This section extends the diabetes choropleth map by adding miniature pie charts on top of each thana. The base map still shows diabetes percentage classes, while the pie charts show the age-group distribution within each thana.

Each miniature pie chart represents two age groups:

  • <=30 years
  • >30 years

The map therefore combines two layers of information:

  1. Background choropleth layer: diabetes patient percentage by thana.
  2. Overlay pie-chart layer: age-group distribution by thana.

The pie charts are shifted slightly above each polygon’s own representative point, while labels remain exactly at the representative point. This prevents labels from drifting into neighboring polygons.

def draw_pie_with_percentages(
    ax,
    x,
    y,
    values,
    colors,
    text_colors,
    radius,
    label_fontsize=6,
    edgecolor="white",
    linewidth=0.50,
):
    """
    Draw a miniature pie chart at a map coordinate
    and place percentage labels inside each slice.
    """
    total = sum(values)

    if total <= 0:
        return

    start_angle = 90

    for value, color, text_color in zip(values, colors, text_colors):
        angle = 360 * value / total
        end_angle = start_angle + angle

        wedge = Wedge(
            center=(x, y),
            r=radius,
            theta1=start_angle,
            theta2=end_angle,
            facecolor=color,
            edgecolor=edgecolor,
            linewidth=linewidth,
            zorder=6,
        )

        ax.add_patch(wedge)

        mid_angle = (start_angle + end_angle) / 2
        mid_angle_rad = np.deg2rad(mid_angle)

        label_x = x + radius * 0.58 * np.cos(mid_angle_rad)
        label_y = y + radius * 0.58 * np.sin(mid_angle_rad)

        label_text = f"{value:.1f}%"

        stroke_color = "white" if text_color == "black" else "black"

        ax.text(
            label_x,
            label_y,
            label_text,
            fontsize=label_fontsize,
            color=text_color,
            ha="center",
            va="center",
            weight="bold",
            zorder=9,
            path_effects=[
                path_effects.withStroke(
                    linewidth=1.2,
                    foreground=stroke_color,
                )
            ],
        )

        start_angle = end_angle

    outline = Circle(
        xy=(x, y),
        radius=radius,
        facecolor="none",
        edgecolor="#222222",
        linewidth=0.50,
        zorder=7,
    )

    ax.add_patch(outline)

## City Corporation

This section visualizes the distribution of diabetes patients across selected thanas of Chattogram City Corporation. The map uses GADM level-3 administrative boundary data from a GeoPackage file and joins it with manually entered thana-level diabetes percentage data.

The spatial boundary file path is loaded from the `.env` file using the `GPKG_PATH` environment variable. This keeps the source code cleaner and avoids hard-coding the data path directly inside the analysis script.

The workflow follows these main steps:

1. Load the required Python libraries.
2. Read the GeoPackage boundary file using `geopandas`.
3. Create a tabular dataset containing diabetes percentages by thana.
4. Clean and standardize thana names for reliable joining.
5. Join the diabetes dataset with the spatial boundary dataset.
6. Classify diabetes percentages into fixed percentage intervals.
7. Apply a red choropleth color palette, starting from a darker red shade.
8. Save the final map as a PDF under the `report/` directory.
9. Preview the generated map directly with `display(fig)`.

The diabetes percentages are classified into the following groups:

- `1–3%`
- `3–5%`
- `5–7%`
- `7–9%`
- `9–11%`
- `11–13%`
- `>13%`

A custom red color palette is used for the choropleth map. Unlike very pale choropleth palettes, the first class begins with a visibly darker red shade so that even low-percentage areas remain clearly visible on the map.

## City Corporation with Age Group Composition

This section extends the diabetes choropleth map by adding miniature pie charts on top of each thana. The base map still shows diabetes percentage classes, while the pie charts show the age-group distribution within each thana.

Each miniature pie chart represents two age groups:

* `<=30 years`
* `>30 years`

The map therefore combines two layers of information:

1. Background choropleth layer: diabetes patient percentage by thana.
2. Overlay pie-chart layer: age-group distribution by thana.

The pie charts are shifted slightly above each polygon’s own representative point, while labels remain exactly at the representative point. This prevents labels from drifting into neighboring polygons.

def draw_pie_with_percentages(
    ax,
    x,
    y,
    values,
    colors,
    text_colors,
    radius,
    label_fontsize=6,
    edgecolor="white",
    linewidth=0.50,
):
    """
    Draw a miniature pie chart at a map coordinate
    and place percentage labels inside each slice.
    """
    total = sum(values)

    if total <= 0:
        return

    start_angle = 90

    for value, color, text_color in zip(values, colors, text_colors):
        angle = 360 * value / total
        end_angle = start_angle + angle

        wedge = Wedge(
            center=(x, y),
            r=radius,
            theta1=start_angle,
            theta2=end_angle,
            facecolor=color,
            edgecolor=edgecolor,
            linewidth=linewidth,
            zorder=6,
        )

        ax.add_patch(wedge)

        mid_angle = (start_angle + end_angle) / 2
        mid_angle_rad = np.deg2rad(mid_angle)

        label_x = x + radius * 0.58 * np.cos(mid_angle_rad)
        label_y = y + radius * 0.58 * np.sin(mid_angle_rad)

        label_text = f"{value:.1f}%"

        stroke_color = "white" if text_color == "black" else "black"

        ax.text(
            label_x,
            label_y,
            label_text,
            fontsize=label_fontsize,
            color=text_color,
            ha="center",
            va="center",
            weight="bold",
            zorder=9,
            path_effects=[
                path_effects.withStroke(
                    linewidth=1.2,
                    foreground=stroke_color,
                )
            ],
        )

        start_angle = end_angle

    outline = Circle(
        xy=(x, y),
        radius=radius,
        facecolor="none",
        edgecolor="#222222",
        linewidth=0.50,
        zorder=7,
    )

    ax.add_patch(outline)
