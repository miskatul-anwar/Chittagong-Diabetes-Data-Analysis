2  Data Cleaning and Standardization

2.1 Overview

This document describes a reproducible data cleaning and standardization workflow for a diabetes patient dataset. The purpose of this workflow is to transform raw patient records into a cleaner and more analysis-ready dataset by removing duplicate fields and records, updating fasting glucose values from a supplementary file, standardizing mixed-format numeric columns, and exporting the cleaned dataset for downstream analysis.

The workflow relies on environment variables to locate the raw dataset, supplementary update file, and final cleaned output path. This approach avoids hard-coding local file locations and makes the pipeline easier to reuse across different environments.

2.2 Objectives

The main objectives of this cleaning process are to:

  • Load the raw diabetes dataset from a configured environment path.
  • Inspect the structure and column inventory of the dataset.
  • Remove duplicate columns created during data collection or export.
  • Remove duplicate patient records using ID as the unique identifier.
  • Update the primary dataset with corrected or newly available fasting glucose values from a supplementary dataset.
  • Convert selected anthropometric variables to numeric format.
  • Extract numeric fasting glucose values from mixed text formats.
  • Remove records that are missing key analytical fields such as Fasting and Age.
  • Export the cleaned dataset to the configured output path.

2.3 Expected Input and Output

This workflow expects the following environment variables to be available:

  • DATASET_PATH: path to the primary raw Excel dataset.
  • NEW_PATH: path to the supplementary Excel dataset containing updated fasting glucose values.
  • CLEANED_DATASET_PATH: path where the cleaned Excel dataset will be saved.

The expected output is a cleaned Excel dataset containing standardized patient records suitable for subsequent analysis, modeling, or reporting.

2.4 Load Dependencies

The workflow begins by importing the required Python libraries. The os module is used to read environment variables, pandas is used for data loading and transformation, and load_dotenv is used to load variables from a .env file.

import os
import pandas as pd
from dotenv import load_dotenv

2.8 Remove Duplicate Columns

Duplicate columns are removed from the dataset. This can happen when Excel files are exported, merged, or manually edited. Keeping only the first occurrence of each column name prevents ambiguity during later transformations.

df = df.loc[:, ~df.columns.duplicated()]

2.9 Remove Duplicate Records

Duplicate patient records are removed using ID as the unique patient identifier. When multiple records share the same ID, the last occurrence is retained. This assumes that later entries are more complete or more recently updated.

df.drop_duplicates(subset="ID", keep="last", inplace=True)

A supplementary dataset is loaded from the path stored in NEW_PATH. This file is used to update the Fasting column in the primary dataset.

Before merging, duplicate records are also removed from the supplementary dataset using the same ID-based rule. Copies of the ID and Fasting columns are created for comparison and verification purposes before the update is applied.

new_file = os.getenv("NEW_PATH")
new_df = pd.read_excel(new_file)
new_df.drop_duplicates(subset="ID", keep="last", inplace=True)

# Create index copies for comparison before merge
df_before_fasting = df[["ID", "Fasting"]].copy()

# Perform merge using ID-based index
df.set_index("ID", inplace=True)
new_df.set_index("ID", inplace=True)
df.update(new_df["Fasting"])
df.reset_index(inplace=True)

2.15 Convert Anthropometric Columns to Numeric Format

The columns BMI, Weight (KG), and Height (CM) are converted to numeric data types. Any non-convertible values are coerced to missing values using errors="coerce".

This step ensures that these fields can be used reliably in later statistical analysis, visualization, or machine learning workflows.

cols = ["BMI", "Weight (KG)", "Height (CM)"]

for col in cols:
    df[col] = pd.to_numeric(df[col], errors="coerce")

2.16 Standardize Fasting Column to Numerical Values

The original Fasting column may contain mixed-format values, such as numbers combined with units or text. A regular expression is used to extract the first numeric value from each entry.

For example, a value such as 10.0MMOL is converted to 10.0. Records without a valid numeric fasting value are treated as missing.

import re   
def extract_numerical_value(value):
    if pd.isna(value):
        return None
    match = re.search(r"(\d+\.?\d*)", str(value))
    return float(match.group(1)) if match else None
df["Fasting"] = df["Fasting"].apply(extract_numerical_value)
df["Fasting"] = df["Fasting"].astype(float)
df.dropna(subset=["Fasting"], inplace=True)

2.17 Verify Numerical Standardization

This verification step confirms how many records have valid numerical fasting values after extraction and type conversion. It also reports the number of missing fasting values and the remaining dataset size.

print(f"Records with missing Fasting: {df['Fasting'].isna().sum()}")

2.18 Remove Records with Missing Age

Records with missing Age values are removed. Age is typically a key variable in diabetes-related analysis, and removing missing values here ensures that downstream analysis can rely on complete age information.

df.dropna(subset=["Age"],inplace=True)

The final dataset is previewed and then exported as an Excel file to the path defined by CLEANED_DATASET_PATH.
